Classes & IDs

Classes

Classes are used to style any amount of elements throuhg css styling. You could style a button in one way and style the button next to it with a different idea.
For example, you can add a class to a button like this:
<button class="testButton">Click me</button>
Then in your CSS, you can style all elements with that class:
(the following would be put in the css sheet) .testButton { background-color: blue; color: white; }

IDs are used to identify an individual elements

For example the sidebar on my website COULD use IDs but I prefer classes for this

<nav id="main-nav"> TextGoesHere</nav> \ The css would look like this. #main-nav { background: #333; }

IDs do communicate with javascript a bit faster though through the "getElementById"

Pros and cons of each

Classes

  • ✓ Efficient for applying the same style to multiple elements
  • ✓ Better for styling
  • ✘ Get overidden by IDs
  • ✘ Can get messy if you solely use classes

IDs

  • ✓ usefull for identifying a stand-alone element you want to style
  • ✓ Selected by Javascript faster than classes
  • ✘ not reusable over pages
  • ✘ not an efficient use with css over classes usually
